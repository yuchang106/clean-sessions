技术栈

⚛️ 前端
  • Next.js 15 (App Router)
  • React 19
  • TypeScript 5
  • TailwindCSS 4
🖥️ 后端
  • Next.js API Routes
  • Node.js fs 模块
  • JSONL 解析/生成
  • 无数据库依赖

页面路由结构

/ ← 首页（会话列表）
/sessions/[id] ← 会话详情页面

API routes:
GET /api/sessions ← 获取所有会话列表
GET /api/sessions/[id] ← 获取单个会话详情
GET /api/sessions/[id]/raw ← 获取原始 JSONL
DELETE /api/sessions/[id] ← 删除会话

组件树

src/app/layout.tsx ← 根布局
src/app/page.tsx ← 首页（会话列表）
├── components/SessionStats.tsx ← 统计卡片
├── components/SessionFilter.tsx ← 项目筛选
├── components/SessionTable.tsx ← 会话表格
├── components/Pagination.tsx ← 分页
└── components/DeleteDialog.tsx ← 删除确认弹窗

src/app/sessions/[id]/page.tsx ← 会话详情页
├── components/SessionHeader.tsx ← 会话信息头部
├── components/MessageBubble.tsx ← 消息气泡
├── components/MessageFilter.tsx ← 消息筛选
└── components/RawJsonViewer.tsx ← 原始JSONL查看器

数据流

列表页数据流:
1. Page 加载 → fetch('/api/sessions')
2. API Route 读取 ~/.claude/history.jsonl
3. 解析 JSONL，按 sessionId 聚合，取最新 display
4. 返回排序后的会话列表 JSON
5. Page 渲染 SessionTable 组件

详情页数据流:
1. Page 加载 → fetch('/api/sessions/[id]')
2. API Route 查找 ~/.claude/projects/*/[id].jsonl
3. 解析 JSONL 逐行，提取消息序列
4. 返回消息列表 JSON（含类型、内容、时间戳）
5. Page 渲染 MessageBubble 列表

删除数据流:
1. 点击删除 → DeleteDialog 确认弹窗
2. 确认 → DELETE /api/sessions/[id]
3. API Route 依次：
a) 删除 projects/*/[id].jsonl
b) 删除 session-env/[id]/
c) 读取 history.jsonl → 过滤掉该 sessionId 的行 → 重写文件
4. 返回删除结果 → 刷新列表

不含数据库，直接操作文件系统，轻量高效
